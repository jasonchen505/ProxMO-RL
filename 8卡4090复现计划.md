# ProxMO-RL 8卡4090复现计划

## 一、资源评估与可行性分析

### 1.1 硬件资源对比

| 资源 | 论文配置 | 你的配置 | 差异分析 |
|------|----------|----------|----------|
| **GPU** | 8×A100 (80GB) | 8×4090 (24GB) | 显存减少70%，需要调整并行策略 |
| **CPU** | 未明确 | AMD EPYC 7763 64核 | 充足 |
| **内存** | 未明确 | 503GB | 充足 |

### 1.2 4090 vs A100 关键差异

```
┌─────────────────────────────────────────────────────────────────┐
│                    GPU规格对比                                   │
├─────────────────────────────────────────────────────────────────┤
│ 指标              │ A100 80GB      │ RTX 4090        │
│───────────────────│────────────────│─────────────────│
│ 显存              │ 80GB HBM2e     │ 24GB GDDR6X     │
│ FP16算力          │ 312 TFLOPS     │ 826 TFLOPS      │
│ 内存带宽          │ 2 TB/s         │ 1 TB/s          │
│ NVLink           │ 支持 (600GB/s) │ 不支持          │
│ 多GPU通信        │ NVLink高速     │ PCIe (64GB/s)   │
└─────────────────────────────────────────────────────────────────┘

关键限制:
1. 显存: 24GB vs 80GB → 需要更小的batch或更多GPU并行
2. 通信: PCIe vs NVLink → 多卡通信效率较低
3. 优势: 4090单卡算力更高，适合数据并行
```

### 1.3 内存需求估算

**7B模型训练内存需求**:
```
模型参数: 7B × 2 bytes (BF16) = 14 GB
梯度: 14 GB
优化器状态 (AdamW): 14 × 2 = 28 GB (fp32 copy + momentum)
激活值: 取决于batch_size和序列长度
─────────────────────────────────────
总计: ~56 GB + 激活值

4090 24GB限制:
- 无法单卡训练7B模型
- 需要FSDP分片 + gradient checkpointing
- 可能需要optimizer offload到CPU
```

**1.5B模型训练内存需求**:
```
模型参数: 1.5B × 2 bytes (BF16) = 3 GB
梯度: 3 GB
优化器状态: 3 × 2 = 6 GB
─────────────────────────────────────
总计: ~12 GB + 激活值

4090 24GB限制:
- 可以单卡训练（需要gradient checkpointing）
- 更适合4090配置
```

### 2.2 配置调整方案

**方案A: 1.5B模型 (推荐)**
```bash
# 4090优化配置
MODEL_PATH="Qwen2.5-1.5B-Instruct"
n_gpus_per_node=8
tensor_model_parallel_size=2        # 2卡一个模型实例
gpu_memory_utilization=0.75         # 留一些余量
train_batch_size=8                  # 减小batch size
group_size=8                        # 保持group size
ppo_micro_batch_size_per_gpu=8      # 减小micro batch
max_prompt_length=2048              # 保持
max_response_length=512             # 保持
enable_gradient_checkpointing=True  # 必须开启
param_offload=False                 # 1.5B不需要
optimizer_offload=False             # 1.5B不需要
```

**方案B: 7B模型 (需要更多调整)**
```bash
# 4090优化配置
MODEL_PATH="Qwen2.5-7B-Instruct"
n_gpus_per_node=8
tensor_model_parallel_size=4        # 4卡一个模型实例
gpu_memory_utilization=0.70         # 更保守
train_batch_size=4                  # 更小batch size
group_size=4                        # 减小group size
ppo_micro_batch_size_per_gpu=4      # 更小micro batch
max_prompt_length=1024              # 减小序列长度
max_response_length=256             # 减小响应长度
enable_gradient_checkpointing=True  # 必须开启
param_offload=True                  # 7B需要
optimizer_offload=True              # 7B需要
```

### 3.2 快速验证: 最小化配置 (Day 1-2)

#### Step 2.1: 创建4090优化的训练脚本
```bash
# 创建新的训练脚本
cat > examples/proxmo_trainer/run_alfworld_4090.sh << 'EOF'
#!/usr/bin/env bash
set -euxo pipefail

project_name='proxmo_alfworld_4090'
exp_name='proxmo_qwen2.5_1.5b_4090'

ENGINE=${1:-vllm}
export VLLM_ATTENTION_BACKEND=XFORMERS

RAY_DATA_HOME=${RAY_DATA_HOME:-"${HOME}/.cache/model_data"}

export HF_HOME=${RAY_DATA_HOME}/.cache/huggingface
export HF_DATASETS_CACHE=${RAY_DATA_HOME}/.cache/huggingface/datasets
export TRANSFORMERS_CACHE=${RAY_DATA_HOME}/.cache/huggingface/transformers

MODEL_PATH=${MODEL_PATH:-"${RAY_DATA_HOME}/models/Qwen2.5-1.5B-Instruct"}
CKPTS_DIR=${CKPTS_DIR:-"${RAY_DATA_HOME}/ckpt/${project_name}/${exp_name}"}

# 4090优化配置
train_data_size=4               # 最小化验证
val_data_size=16                # 最小化验证
group_size=4                    # 减小group size
mode="soft_grouping"
num_cpus_per_env_worker=0.1

max_prompt_length=1024          # 减小序列长度
max_response_length=256         # 减小响应长度

n_gpus_per_node=2               # 先用2卡验证
NNODES=1

total_epochs=5                  # 最小化验证
save_freq=5
test_freq=2
val_before_train=True

# 推理配置
temperature=0.4
do_sample=True
tensor_model_parallel_size=1    # 1.5B模型单卡推理
gpu_memory_utilization=0.75

ppo_mini_batch_size=32          # 减小batch size
ppo_micro_batch_size_per_gpu=8  # 减小micro batch
log_prob_micro_batch_size_per_gpu=8

python3 -m verl.trainer.main_ppo \
    algorithm.adv_estimator=proxmo \
    data.train_files="${TRAIN_FILE}" \
    data.val_files="${TEST_FILE}" \
    data.train_batch_size=${train_data_size} \
    data.val_batch_size=${val_data_size} \
    data.max_prompt_length=${max_prompt_length} \
    data.max_response_length=${max_response_length} \
    data.filter_overlong_prompts=True \
    data.truncation='error' \
    data.return_raw_chat=True \
    actor_rollout_ref.model.path="${MODEL_PATH}" \
    actor_rollout_ref.actor.optim.lr=1e-6 \
    actor_rollout_ref.model.use_remove_padding=True \
    actor_rollout_ref.actor.ppo_mini_batch_size=${ppo_mini_batch_size} \
    actor_rollout_ref.actor.ppo_micro_batch_size_per_gpu=${ppo_micro_batch_size_per_gpu} \
    actor_rollout_ref.actor.use_kl_loss=True \
    actor_rollout_ref.actor.kl_loss_coef=0.01 \
    actor_rollout_ref.actor.kl_loss_type=low_var_kl \
    actor_rollout_ref.model.enable_gradient_checkpointing=True \
    actor_rollout_ref.actor.fsdp_config.param_offload=False \
    actor_rollout_ref.actor.fsdp_config.optimizer_offload=False \
    actor_rollout_ref.rollout.log_prob_micro_batch_size_per_gpu=${log_prob_micro_batch_size_per_gpu} \
    actor_rollout_ref.rollout.tensor_model_parallel_size=${tensor_model_parallel_size} \
    actor_rollout_ref.rollout.name=${ENGINE} \
    actor_rollout_ref.rollout.gpu_memory_utilization=${gpu_memory_utilization} \
    actor_rollout_ref.rollout.enable_chunked_prefill=False \
    actor_rollout_ref.rollout.enforce_eager=False \
    actor_rollout_ref.rollout.free_cache_engine=False \
    actor_rollout_ref.rollout.val_kwargs.temperature=${temperature} \
    actor_rollout_ref.rollout.val_kwargs.do_sample=${do_sample} \
    actor_rollout_ref.ref.log_prob_micro_batch_size_per_gpu=${log_prob_micro_batch_size_per_gpu} \
    actor_rollout_ref.ref.fsdp_config.param_offload=False \
    actor_rollout_ref.actor.use_invalid_action_penalty=True \
    actor_rollout_ref.actor.invalid_action_penalty_coef=0.1 \
    algorithm.use_kl_in_reward=False \
    algorithm.gamma=0.95 \
    algorithm.proxmo.step_advantage_w=1.0 \
    algorithm.proxmo.mode=${mode} \
    +algorithm.proxmo.enable_psc=True \
    +algorithm.proxmo.psc_alpha=4.0 \
    +algorithm.proxmo.psc_target_range=0.05 \
    env.env_name=alfworld/AlfredTWEnv \
    env.seed=0 \
    env.max_steps=20 \
    env.rollout.n=${group_size} \
    env.resources_per_worker.num_cpus=${num_cpus_per_env_worker} \
    trainer.critic_warmup=0 \
    trainer.logger=['console','tensorboard'] \
    trainer.project_name="${project_name}" \
    trainer.experiment_name="${exp_name}" \
    trainer.n_gpus_per_node=${n_gpus_per_node} \
    trainer.nnodes=${NNODES} \
    trainer.save_freq=${save_freq} \
    trainer.test_freq=${test_freq} \
    trainer.total_epochs=${total_epochs} \
    trainer.default_local_dir="${CKPTS_DIR}" \
    trainer.ray_wait_register_center_timeout=600 \
    trainer.val_before_train=${val_before_train} $@
EOF

chmod +x examples/proxmo_trainer/run_alfworld_4090.sh
```

## 六、常见问题与解决方案

### 6.1 OOM (Out of Memory)

**症状**: CUDA out of memory error

**解决方案**:
```bash
# 1. 减小batch size
train_batch_size=4
ppo_micro_batch_size_per_gpu=4

# 2. 减小序列长度
max_prompt_length=1024
max_response_length=256

# 3. 增加GPU并行
tensor_model_parallel_size=4

# 4. 启用offload
param_offload=True
optimizer_offload=True

# 5. 减小group size
group_size=4
```

### 6.2 训练速度慢

**症状**: 每步耗时过长

**解决方案**:
```bash
# 1. 增加batch size（如果内存允许）
ppo_micro_batch_size_per_gpu=32

# 2. 使用更高效的推理引擎
ENGINE=vllm

# 3. 启用chunked prefill
enable_chunked_prefill=True

# 4. 调整GPU内存利用
gpu_memory_utilization=0.85
```

### 6.3 环境相关问题

**症状**: ALFWorld环境初始化失败

**解决方案**:
```bash
# 1. 重新下载资源
alfworld-download -f

# 2. 检查Java版本（ALFWorld需要Java）
java -version

# 3. 测试环境
alfworld-play-tw
```

### 6.4 Ray集群问题

**症状**: Ray启动失败或worker崩溃

**解决方案**:
```bash
# 1. 清理Ray状态
ray stop
rm -rf /tmp/ray

# 2. 增加超时时间
trainer.ray_wait_register_center_timeout=1200

# 3. 减少worker数量
n_gpus_per_node=4
```
